# Network Effect Strategy

## Growth Flywheel

1. Institutions issue trusted credentials on protocol rails.
2. Students demand portable verification acceptance across platforms.
3. Employers and verifiers prefer low-friction trusted checks.
4. Verified demand attracts additional institutions and issuers.

## Execution Levers

- Open API incentives:
  - discounted usage tiers for high-volume compliant integrators
  - rewards for verification contribution quality
- Integration grants:
  - milestone-based grants from treasury grants bucket
  - institution LMS/HRIS connector subsidies
- SDK distribution strategy:
  - official SDK package (`sdk/vindicate-js`)
  - implementation templates and OpenAPI contract
- Developer ecosystem rewards:
  - contribution scoring + grants + reputation badges
  - hackathon and extension module bounties

## Network Optimization Rules

- Prioritize integrations that maximize credential portability and verifier demand.
- Tie emissions to verified usage growth, not wallet count growth.
- Penalize low-quality issuance to preserve trust quality of network effects.
